1. Xperior Library¶

Summary

Info:concept of additional configurations, features and services. It is recommended to use existing code to avoid reinventing the wheel. Other modules similar to xlib can and are presented in xlib as well, extending such code and structuring it into groups.
Target:users
Status:mature

Functions, procedures, classes, etc. - generally objects coded in xlib.py file, comes with version of XPERIOR. Users are encouraged to use xlib’s objects in test scripts itself to simplify and improve the coding. The backward compatibility of objects defined in xlib is mather of discipline of file developers

xlib.copy_remote_file_from(device, from_path, to_path)[source]¶

Copy remote file from VISA interface -> None.

Device specifies remote device. Both paths are absolute file paths. If file on device doesn’t exist, “to_path” file content will carry error message.

Note:
  • path names like: r’parentchild’ or ‘parent\child’
  • File size is limited to 999.999999 MB.
  • Use remote device’s ‘D:’ drive (user guide recommendation).
xlib.copy_remote_file_to(device, from_path, to_path)[source]¶

Copy remote file to VISA interface -> None.

Device specifies remote device. Both paths are absolute file paths. IOError exception will be raised, if file on “from_path” doesn’t exist.

Note:
  • path names like: r’parentchild’ or ‘parent\child’
  • File size is limited to 999.999999 MB.
  • Use remote device’s ‘D:’ drive (user guide recommendation).
xlib.correlation(pattern1, pattern2, tolerance=0)[source]¶

Binary Correlation Function -> list of “K”s.

both arguments have to be either strings or lists of numbers. If numbers, absolute tolerance can be specified with ‘tolerance’ keyword. If ‘pattern1’ is larger than ‘pattern2’, patterns are switched. Function returns list of K[%]. K is instantaneous correlation value according to the formula:

Matches - NonMatches
K[%] = ———————- * 100
Matches + NonMatches

+===================================================+ | pattern 2 | +===========+=======================================+ | pattern 1 | +===========+ +===========+ | | pattern 1 | | correlation shifts +===========+ |—————————————|

xlib.create_panel(type, name, show=True)[source]¶

Creates a panel parented to AUI FrameManager of the app -> panel instance.

Get certain type of Frame which can be used for general purpose. Such a frame can be used in script as well as in Shell.

Available frame types are:
“” .. just a simple frame “TextCtrl” .. with TextCtrl and write_text and clear_all methods “Plot” .. with Matplotlib Plot Grid

Name variable sets not only the Frame caption but also identifies the Frame itself, i.e. calling the function with the same arguments just reactive the already created frame. Note the the same “name” can’t by used for different “type”s.

See examples in libexamples and
libmatplotlib_graph_examples

xlib.execute_stop(reason='')[source]¶

Force Stop of the running script -> None.

It stops in Communicator’s Editor currently running script. It is same as click on ‘Stop’ button or Ctrl+T shortcut.

If multitesting, RuntimeError is raised, which can by catched by single script execution, storing appropriate verdict and starts next test in the queue...

xlib.get_files(top, wildcard='*.*')[source]¶

xlib.get_screen(device, display=True)[source]¶

Takes printscreen of target remote device -> filepath.

File is saved on remote device and transferred to user’s local machine. Local filepath is returned. File is also executed with os.startfile if “display” is set to True

Note: Because of different remote commands is currently supported: FSQ.

xlib.multi_choice_dialog(options_list, text=None)[source]¶

function will be deprecated, please remove it from your scripts.

Commands Editor user dialog -> integer selection.

If text is not given, intern message will be used to inform user to choose item(s) from options_list. If you want to mark multiple items, enter the sum of item’s numbers. Function returns a list of selected items.

Text output is Protocol Trace. Text input is Interactive Console.

Example:

  1. first

2. second 4. third 8. fourth ... type 14 to select last 3 items.

xlib.permutation(list)[source]¶

Permutation -> list.

Takes a list of objects and returns a generator yielding all permutations.

Example for clarity:

list(permutation([1,2,3])) returns: [[1, 2, 3],

[2, 1, 3], [2, 3, 1], [1, 3, 2], [3, 1, 2], [3, 2, 1]]

Plot a graph -> plot instance.

Plot a single graph and displays the result in separate window. See plot __doc__ at xlib.pltframe.plot.

Example: plot([x*x for x in xrange(20)])

xlib.report_off()[source]¶

xlib.report_on()[source]¶

xlib.sdir(obj, string='', ignorecase=True, list_print=True, nobaseclass=False)[source]¶

sdir -> list of strings.

Returns list of attributes of specified object. Optionally only these containing ‘string’ with or without case sensitivity. Optionally will every element of the list will be printed at the new line. Optionally only these which are not contained in the base class of the obj (baseclass reference stored in __base__ attribute).

xlib.set_system_path(absolute_path_without_drive_letter)[source]¶

Appends path to sys.path -> bool success.

It will be searched in range of all drivers + absolute_path_without_drive_letter. If absolute path exists, it will be appended to sys.path.

xlib.show_last_error()[source]¶

Shows last error logging traceback -> None.

xlib.shutdown()[source]¶

Switches the local machine OFF -> bool status.

Also executes XPERIOR’s OnClose action.

For security reasons the flag ‘_int.modules.control.ENABLE_SHUTDOWN’ has to be set to True.

xlib.shutdown_on_stop(enable=True)[source]¶

Shutdown the PC when script in Commands Editor stops -> bool status.

If ‘enable’ keyword is set to False, all shutdown procedures will be discarded. See shutdown function at ‘_int.control.shutdown()’ for more info.

xlib.stop_on_visa_status(instrument, periodically=True)[source]¶

Stops on remote device timeout(s) -> None (throw exception).

If Visa Error Status (VES) is detected, program execution will raise an exception. On next script start, execution stops on first occurrence of VES again if “periodically” is True; after 10 otherwise.
